# Codexia Development Workflow

## Purpose

Defines the preferred process for developing Codexia itself.


## Workflow

1. Understand existing architecture

2. Identify affected systems

3. Define intended behaviour

4. Plan implementation

5. Make the smallest appropriate change

6. Run validation

7. Review results

8. Commit changes


## Principles

Prefer:

- incremental evolution
- explicit reasoning
- small changes
- validated outcomes


Avoid:

- large rewrites
- unnecessary abstraction
- bypassing existing systems
