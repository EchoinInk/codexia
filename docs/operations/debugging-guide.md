# Codexia Debugging Guide

## Purpose

Common troubleshooting approaches for Codexia systems.


## Planner Issues

Check:

- planner schema
- model output parsing
- fallback handling
- tool selection


## Workflow Issues

Check:

- state transitions
- executor results
- validator expectations


## Intelligence Issues

Check:

- indexing state
- parser behaviour
- dependency relationships


## General Debugging Process

1. Reproduce issue

2. Identify responsible subsystem

3. Inspect state transitions

4. Validate assumptions

5. Fix smallest affected area

6. Add regression coverage
